Glidix Memory Mapping

All virtual memory available to a Glidix process is said to be mapped into its address space. Hence, the address space is a list of mappings, each with different permissions, locations and types. Under Glidix, the following types of mappings exist:

  • Anonymous mapping - memory used privately by a single process, backed by physical memory but no filesystem objects. When first accessing a page of an anonymous mapping, the page is filled with zeroes by the system.
  • Device memory mapping - a mapping of memory backed on a separate device (not RAM). For example, this may refer to a graphics card's video memory.
  • Private file mapping - backed by physical memory, initialized with data from a file. The kernel attempts to minimize the number of duplications of a file's data in memory. When first accessing a page from such a mapping, it is filled with the file data. Modifying the data does not write it back to the file.
